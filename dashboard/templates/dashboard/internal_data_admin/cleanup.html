Safer clutter removal

Clean campaigns through one or more reporting systems

You should not delete from every table by hand. Use batch cleanup when campaign IDs span Inclinic, SAPA/RFA, and PE; use single cleanup for a targeted one-campaign repair.

Delete order Execution runs downstream first: GOLD -> SILVER -> BRONZE -> RAW. The whole cleanup is one transaction and one audited operation per table.

What to delete for what

Permanent dummy/test cleanup

Choose RAW so the source-copy rows and all derived BRONZE/SILVER/GOLD records are cleared together. Rerun the system ETL after source cleanup.

Bad derived output

Choose BRONZE or SILVER when source rows are valid but derived/reporting rows need a clean rebuild. Choose GOLD only for stale dashboard output.
